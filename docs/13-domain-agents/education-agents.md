# 教育・学習支援エージェントの設計

## この記事の目的

「答えを渡す」のではなく「**学習を進める**」エージェントを、教育設計と安全の両面から作れるようになります。段階的ヒント・学習者への適応・誤答への向き合い方・未成年や依存への配慮・学習効果の測定の難しさ・教育現場の規制の確認先を、実務の設計判断として持ち帰れる状態を目指します。

## 対象読者

- 学習支援・チュータリング・社内研修などの教育系エージェントを設計するエンジニア・プロダクト責任者
- 「賢く答える」ことと「学びを促す」ことの違いを、設計に落としたい実装者

- [ガードレール](../06-security/guardrails.md) — 出力・行動の外側からの制御(安全設計の基盤)
- [信頼度と較正](../04-evaluation/confidence-and-calibration.md) — 確信度の扱い(誤答・不確かさへの対応)

## 本文

### 概要: 目的関数が「正解」ではなく「学び」

多くの LLM アプリは「正しく答える」ことを目指します。教育エージェントは、ここが根本的に違います。**目的は学習者が理解し、できるようになること**であり、答えを即座に渡すことは、しばしばその目的に反します。答えを教えれば課題は解けますが、学習者は学べません。

したがって設計の中心は「**どこまで教え、どこで待つか**」の設計です。速く正解を出す最適化とは逆で、あえて答えを渡さず、考えさせる余地を残します。以下の各節は、この「学びを促す」ための判断に収束します。同時に、対象に未成年を含みうるため、**安全の設計が機能と同格**である点も、他ドメインと違う特徴です。

### 教育設計の原則(段階的ヒント・生産的なもがき)

「学びを進める」を具体化する定番が、答えを一度に渡さず、**段階的に導く**設計です。

- **段階的ヒント**: いきなり答えを出さず、「まずどこに着目するか」「次に何を試すか」と、**ヒントの粒度を段階的に**上げます。学習者が自力で進めるなら、そこで止めます
- **ソクラテス式に問い返す**: 答えを与える代わりに、考えを引き出す問いを返します。「なぜそう考えたか」「この場合はどうなるか」と問い、学習者自身に気づかせます
- **生産的なもがきを守る**: 学習者が少し行き詰まる状態(productive struggle)は、学びに必要です。**すぐに助け船を出しすぎない**ことが、かえって学習効果を高めます。もがきを奪わない介入の設計が、教育エージェント固有の難しさです
- **ただし詰まらせすぎない**: もがきが「もう無理」に変わる前に助けます。この見極めが、次節の学習者モデルにつながります

### 学習者モデル(理解度の推定と適応)

一律の教え方は、教育では機能しません。学習者ごとに理解度が違うため、**推定して適応**します([長期記憶の実装](../03-implementation/long-term-memory-implementation.md)の応用)。

- **理解度を推定する**: 学習者の応答・誤り方・つまずく箇所から、どこまで理解しているかを推定します。既に分かっていることを繰り返せば退屈させ、飛ばしすぎれば置いていきます
- **適応の記憶を持たせる**: 「この学習者はここが苦手」「この説明で通じた」を記憶し、次の対話で活かします。ただし個人の学習データは機微であり、プライバシー([プライバシー強化技術](../06-security/privacy-enhancing-technologies.md))と保護者・本人の同意の対象として扱います
- **推定を過信しない**: 理解度の推定は外れます。推定に固執せず、学習者の反応で更新します。断定的な決めつけ(「あなたはこれが苦手」)は、学習意欲を削ぐため避けます

### 誤答への向き合い(訂正の仕方・自信の扱い)

学習者は間違えます。誤答への向き合い方が、学習体験の質を大きく左右します。

- **誤りを責めず、手がかりにする**: 誤答は、どこで理解がずれているかを教えてくれます。「違います」で終えず、**なぜその誤りが起きたか**を推し量り、そこに合わせて導きます
- **エージェント自身の不確かさを扱う**: 教育エージェント自身も間違えます。**自信のない内容を断定しない**ことが、特に教育では重要です([信頼度と較正](../04-evaluation/confidence-and-calibration.md))。誤った知識を自信満々に教えると、学習者は誤りごと覚えます。確度が低ければ「一緒に確かめよう」と一次資料へ導きます
- **迎合を避ける**: 学習者に気に入られようと、誤った答えに同調したり、できていないのに褒めたりするのは有害です。優しさと正確さを両立させます

### 安全(未成年への配慮・誤情報・依存)

教育エージェントは、対象に未成年を含みうるため、安全の設計を機能と同格に扱います。[ガードレール](../06-security/guardrails.md)を出力・行動の外側に置きます。

- **未成年への配慮**: 年齢に不適切な内容・危険な助言をブロックする[ガードレール](../06-security/guardrails.md)を設けます。これはプロンプトのお願いではなく、コードで強制する制御にします
- **誤情報を教えない**: 教育での誤情報は、学習者が誤りごと覚えるため被害が長期化します。事実性が重要な領域では、根拠の提示・一次資料への誘導を組み込みます
- **依存・過剰信頼を防ぐ**: 「エージェントに聞けば全部答えてくれる」への依存は、自力で学ぶ力を損ないます。答えを渡しすぎない設計(段階的ヒント)は、依存対策でもあります
- **人の教育者との接続**: 重要な判断(評価・進路・メンタル面の兆候)は、エージェントで完結させず**人の教育者へエスカレーション**します([Human-in-the-Loop 設計](../02-architecture/human-in-the-loop.md))

### 評価(学習効果をどう測るかの難しさ)

教育エージェントの評価は、他ドメインより難しく、注意が要ります。**「正しく答えたか」と「学習者が学べたか」は別物**だからです。

- **正答率は目的ではない**: エージェントが正しい説明をしたかは測れますが、それは目的ではありません。目的は学習者の理解が進んだかで、これは 1 回の対話では測れません
- **代理指標を慎重に使う**: つまずきからの回復・次の類題の正答・自力で進めた割合などを代理指標にします。ただし代理指標の最適化が本来の学習を損なわないか(答えを教えて短期の正答率だけ上げる、など)を監視します
- **学習効果は長期で見る**: 本当の効果は、後日の定着・応用で表れます。短期の満足度や正答だけで判断せず、可能なら長期の学習成果と突き合わせます。教育効果の測定は教育学の専門領域でもあり、**測りにくさ自体を前提に**設計します

### 教育現場の規制・ガイドライン

教育、特に初等中等教育や公的機関での利用は、規制・ガイドラインの対象になりえます。本記事は内容を解説せず、**確認先の所在**を示します。

- **確認先を押さえる**: 教育分野での AI 利用には、各国・各自治体・教育機関のガイドラインが関わりえます。適用の有無・内容は[業界別規制の入口マップ](../09-business/industry-regulations-map.md)の教育の項を入口に、一次情報で確認します
- **子どものデータ保護**: 未成年の個人データには固有の保護要件がかかりえます。学習データの収集・保存・利用は、保護者・本人の同意と規制の両面で確認します(本記事は確認先の提示までで、法的判断は専門家の領分です)

## 実務での注意点

### アンチパターン

- **問われたら即座に答えを渡す** → 課題は解けても学習者は学べない → 段階的ヒント・問い返しで、自力で進める余地を残す
- **助け船を出しすぎる** → 生産的なもがきを奪い学習効果が下がる → すぐ助けず、行き詰まりが限界に達する前に介入する
- **一律の教え方をする** → 分かっている人を退屈させ、追いつけない人を置いていく → 理解度を推定して適応する
- **エージェントが自信のない内容を断定する** → 学習者が誤りごと覚える → 確度が低ければ断定せず一次資料へ導く
- **学習者に迎合して褒める・同調する** → できていないのに満足させ学びを妨げる → 優しさと正確さを両立させる
- **正答率で学習効果を測ったつもりになる** → 短期の正答が学びとすり替わる → 代理指標を慎重に使い、長期の定着で見る
- **未成年の安全・データ保護をプロンプト任せにする** → 不適切な内容・データ漏えいのリスク → ガードレールで強制し、規制の確認先を押さえる

### チェックリスト

- [ ] 答えを即座に渡さず、段階的ヒント・問い返しで自力の余地を残しているか
- [ ] 生産的なもがきを守りつつ、詰まらせすぎない介入の見極めを設計したか
- [ ] 学習者の理解度を推定し、適応する仕組みがあるか(推定を過信しない設計か)
- [ ] エージェント自身の不確かさを扱い、自信のない内容を断定しないか
- [ ] 未成年への不適切内容・危険な助言をガードレールで強制的にブロックしているか
- [ ] 重要な判断を人の教育者へエスカレーションする経路があるか
- [ ] 正答率でなく学習効果(代理指標・長期定着)で評価し、代理指標の最適化の弊害を監視しているか
- [ ] 教育分野の規制・子どものデータ保護の確認先を押さえたか
