# AI の歴史と系譜(深層学習から Agent まで)

## この記事の目的

「深層学習 → Transformer → LLM → エージェント」という系譜を 1 本でつかみ、「**なぜ今こうなっているか**」の文脈を持てるようになります。年表の暗記ではなく、**設計思想の転換点**をたどることで、新しい概念が出てきたときに「これは過去のどの流れの続きか」を位置づけられる状態がゴールです。新規参加者や非専門家への説明にも使えます。

## 対象読者

- LLM・エージェントから入って、それ以前の文脈を短時間でつかみたいエンジニア
- チームの新規参加者・非専門家に「なぜ今こうなっているか」を説明する立場の人

## 前提知識

- [AI Agent とは何か](what-is-an-ai-agent.md) — 本記事が到達する「エージェント」の定義(先読みでも後読みでも可)

## 本文

### 概要: 年表ではなく「転換点」でたどる

AI の歴史は膨大ですが、実務者に効くのは**設計思想がどこで転換したか**です。本記事は網羅的な年表や人物誌ではなく、「何を賢さの源にするか」という発想の転換点を数個たどります。固有のモデル名は、転換点の代表例としてのみ挙げます。

大きな流れは、**「人が知識を書き込む」から「データから学ばせる」、そして「大規模に事前学習した基盤を使い回す」への移行**です。

### 前史: 記号主義と機械学習の往復

初期の AI は、**人間がルールと知識を明示的に書き込む**方向(記号主義・エキスパートシステム)から始まりました。しかし、現実の複雑さをすべてルールで書き切ることはできず、限界に突き当たります。

これに対し、**データから規則性を学ばせる**機械学習が力を得ます。「人が書く」か「データから学ぶ」かの間で重心が往復してきた、というのが前史の要点です。現在の LLM は「データから学ぶ」の極端な発展形ですが、後述するように「人が指示で導く」要素も組み合わさっています。

### 深層学習の転換点

機械学習の中でも、**特徴量を人が設計する**のではなく、**多層のニューラルネットワークが特徴を自分で学ぶ**深層学習が転換点になりました。

- それまでは「何を手がかりにするか(特徴量)」を人が設計していました。深層学習は、この手がかりの発見自体をモデルに任せます
- 画像認識・音声認識で人手設計を大きく上回り、「**規模(データ・計算)を増やすと性能が上がる**」という経験則が見え始めます。これが後のスケーリングの発想につながります

### Transformer と事前学習パラダイム

言語を扱ううえで決定的だったのが、**Transformer** という構造の登場です(その中核である注意機構の仕組みは [注意機構とコンテキストウィンドウの仕組み](../10-llm-foundations/attention-and-context.md))。

- Transformer は、系列の要素間の関係を効率よく捉え、**大規模な並列学習**を可能にしました
- これと組み合わさったのが**事前学習(pre-training)パラダイム**です。大量のテキストで「次に来る語を予測する」だけの汎用的な学習をしておき、後から個別タスクに適応させる、という二段構えです(生成の仕組みは [LLM はどうやってテキストを生成するか](../10-llm-foundations/how-llms-generate-text.md))
- 「タスクごとにモデルを作る」から「**汎用的に事前学習した 1 つのモデルを使い回す**」への転換が、ここで起きました

### スケーリングと基盤モデル化

事前学習の規模を上げ続けると、**予想以上に汎用的な能力**が現れました。

- データ・パラメータ・計算を増やすほど性能が上がる、という傾向が確認され、規模の拡大が競争の中心になります
- こうして生まれた、多様なタスクに使い回せる大規模モデルが**基盤モデル(foundation model)**です。「1 つの基盤を、多くの用途に適応させる」という現在の構図が確立します
- 学習工程の全体像(事前学習 → 指示チューニング → 選好調整)は [LLM の学習パイプライン(事前学習から選好調整まで)](../10-llm-foundations/llm-training-pipeline.md) が扱います

### 指示追従・対話化

事前学習しただけのモデルは「続きを生成する」だけで、人の指示に素直には従いません。ここに**指示追従・対話化**の工程が加わりました。

- 「指示 → 望ましい応答」の例で追加学習し(指示チューニング)、さらに人の好みを反映させる(選好調整)ことで、**対話して指示に従う**モデルになります
- この転換で、AI は研究者の道具から、**誰もが自然言語で使える道具**になりました。同時に、迎合や過剰な拒否といった癖もこの工程に由来します([LLM の学習パイプライン](../10-llm-foundations/llm-training-pipeline.md))

### ツール使用とエージェント化

対話できる LLM に、**外部の道具を使わせ、ループで動かす**ことで、エージェントが生まれました。

- LLM は単体では計算・検索・実世界の操作ができません。**ツール使用**([ツール使用](tool-use.md))で外部機能を呼べるようにし、**ループ**([Agent ループ](agent-loop.md))で「観測 → 思考 → 行動」を繰り返させることで、目標に向けて自律的に動く**エージェント**([AI Agent とは何か](what-is-an-ai-agent.md))になります
- 「1 回の応答を返す道具」から「**手順を自分で決めて実行するシステム**」への転換が、現在の到達点です。この延長線上に、物理世界で動く個体([フィジカル AI とロボティクスの概観](physical-ai-overview.md))もあります

### 歴史から学べる判断の教訓

系譜を知る実務上の価値は、**流行への向き合い方**にあります。

- **ハイプと実用の距離を読む**: AI は過熱(ハイプ)と幻滅を繰り返してきました。派手なデモが出たとき、「これは過去のどの転換点の延長か」「実用まで何が足りないか」を歴史の視点で読むと、過剰な期待にも過剰な悲観にも振られにくくなります
- **転換の本質は「賢さの源の移動」**: 人が書く → データから学ぶ → 基盤を使い回す、という移動が繰り返されてきました。新しい概念も、この軸のどこにあるかで位置づけられます
- **能力予測は断定しない**: 「いつ何ができるようになるか」の予測は外れ続けてきました。系譜は「なぜ今こうか」を説明しますが、未来の断定には使えません

## 実務での注意点

### アンチパターン

- **年表・固有名詞の暗記に走る** → モデル名を覚えても判断に効かず、すぐ陳腐化する → 「賢さの源がどう移ったか」の転換点で理解する
- **新概念を歴史から切り離して受け取る** → 「まったく新しい魔法」と誤認し、過剰な期待をする → 過去のどの流れの続きかで位置づける
- **デモの派手さで実用度を判断する** → ハイプに振られ、投資判断を誤る → 「実用まで何が足りないか」を歴史の視点で読む
- **系譜を能力予測に使う** → 「この延長で必ずこうなる」と断定する → 系譜は「なぜ今こうか」の説明に留め、未来は断定しない
