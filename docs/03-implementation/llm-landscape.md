# 主要 LLM の全体像(モデルカタログ)

## この記事の目的

主要プロバイダー(Anthropic / OpenAI / Google)とオープンウェイト系の現行モデルファミリーについて、ティア構成・特性・コスト帯・使用場面を一覧できるようになります。「どう選ぶか」の判断フレームは [モデル選定ガイド](model-selection.md) が担当し、本記事はその具体(地図)を提供します。

**本記事はこのライブラリで最も情報の鮮度リスクが高いページです。** モデルの顔ぶれ・価格は月単位で変わるため、意思決定の際は必ず本文冒頭の最終確認日と各公式ページを確認してください。

### 地図の読み方: 各社に共通する構造

2026-07 時点の主要 3 社は、驚くほど似た構造に収斂しています。この共通構造を先に押さえると、個々のモデル名に振り回されなくなります。

- **3 ティア構成**: 上位(最高性能)/ 中位(バランス・本番の主力)/ 軽量(速く安い)のファミリーを各社が持ちます
- **推論(thinking / reasoning)はモード統合**: 「推論専用の別モデル」は縮小し、汎用モデルの思考の深さをパラメータ(effort 等)で制御する形が標準になりました
- **1M 級コンテキストと画像入力が標準装備**(中位以上)。テキスト出力が基本で、音声・画像生成は別系統のモデルが担当します
- **料金構造も共通**: 出力は入力の 5〜6 倍、キャッシュ読み取りは入力の約 1 割、バッチ処理は半額(いずれも 2026-07 時点の傾向)

差が出るのは、**モダリティの幅(動画・音声入力)/ 長コンテキストの割増の有無 / 提供経路とガバナンス / オープンウェイトかどうか**です。

### Anthropic(Claude ファミリー)

| ティア | モデル(2026-07 時点) | 位置づけ(公式表現の要旨) |
| --- | --- | --- |
| フロンティア | Claude Fable 5 | 最も要求の厳しい推論・長時間エージェント向けの最上位(常時 adaptive thinking) |
| 上位 | Claude Opus 4.8 | 複雑なエージェンティックコーディングとエンタープライズ業務。「迷ったらまずこれ」と公式が案内 |
| 中位 | Claude Sonnet 5 | 速度と知能のバランス。本番ワークロードの大半 |
| 軽量 | Claude Haiku 4.5 | ニアフロンティア知能の最速・最安モデル |

- **特性**: 中位以上は 1M トークン入力 / 128K 出力(Haiku は 200K / 64K)。画像入力対応(動画・音声入力は非対応)。**1M コンテキストに長文割増がない**のは 3 社の中で特徴的です
- **コスト帯**(2026-07 時点の桁感): 入力単価は軽量からフロンティアまで約 10 倍の幅(おおよそ $1〜$10 / 1M トークン)、出力はその約 5 倍。Sonnet 5 は 2026-08 末まで導入価格
- **使用場面**: 公式指針は「単純なタスクは Haiku、本番の大半は Sonnet、最も複雑な推論は Opus、最高能力が必要なら Fable 5」
- **補足**: 退役は 60 日以上前に通知するポリシーが明文化されています。Fable 5 は安全分類器による拒否(refusal)応答の設計と 30 日データ保持の必須化という固有の運用条件があります。現行世代の一部(Opus 4.7 以降の Opus・Sonnet 5・Fable 5)は新トークナイザで**同じテキストが約 3 割多くトークン化される**ため、旧世代からの移行時はコスト見積りの再計算が必要です

### OpenAI(GPT ファミリー)

| ティア | モデル(2026-07 時点) | 位置づけ(公式表現の要旨) |
| --- | --- | --- |
| フラッグシップ | GPT-5.5 | コーディング・プロフェッショナルワーク向けの最新世代 |
| 高コンピュート | GPT-5.5-pro / 5.4-pro | 数分かけてでも最高品質の回答が必要な難問向け |
| 低価格フラッグシップ | GPT-5.4 | 5.5 と同系の能力をより低コストで |
| mini | GPT-5.4-mini | 軽量コーディング・サブエージェント |
| nano | GPT-5.4-nano | 分類・抽出・ランキングなど速度とコスト最優先 |

- **特性**: 5.5 / 5.4 は約 1M(mini / nano は 400K)入力 / 128K 出力。画像入力対応。推論は effort 5 段階(none〜xhigh)で制御し、**272K トークン超の入力には割増**(入力 2 倍・出力 1.5 倍)があります。リアルタイム音声(gpt-realtime 系)・画像生成(gpt-image 系)・動画生成(sora 系)は別系統です
- **コスト帯**(2026-07 時点の桁感): 入力単価は nano から 5.5 まで約 25 倍の幅(おおよそ $0.2〜$5 / 1M トークン)、出力はその約 6 倍。pro 系はさらに約 1 桁高い別格の価格帯です
- **使用場面**: 公式の選定ガイドは「**まず GPT-5.5 で精度目標を達成し、その後に精度を保てる最も小さく速いモデルへ置き換える**」という 2 段階アプローチ。effort の上げすぎによる過剰思考(overthinking)の害を公式自身が警告しています
- **補足**: かつての推論特化「o シリーズ」の o3 / o3-pro は 2026-12 に退役予定で、公式の移行先は GPT-5.5 系です(推論は専用モデルではなく本体の effort 制御に集約された形)。ChatGPT 追随の `chat-latest` は本番利用非推奨(中身が随時変わる)と公式が明記しています

### Google(Gemini ファミリー)

| ティア | モデル(2026-07 時点) | 位置づけ(公式表現の要旨) |
| --- | --- | --- |
| 上位 | Gemini 3.1 Pro(**プレビュー**) | 高度な問題解決。正確なツール使用が要るエージェント・ソフトウェアエンジニアリング |
| 主力 | Gemini 3.5 Flash(安定版) | フロンティア級知能を高速・低コストで。サブエージェント・多段ワークフローの主力 |
| 中位(プレビュー) | Gemini 3 Flash(プレビュー) | マルチモーダル理解に強みを持つ、と公式が説明 |
| 軽量 | Gemini 3.1 Flash-Lite(安定版) | 大量・低遅延処理、単純な抽出・翻訳、モデルルーティング |
| 移行期の旧世代 | Gemini 2.5 Pro / Flash / Flash-Lite | 2026-10-16 以降に提供終了予定。移行先は 3.x 系 |

- **特性**: 主要モデルは共通して 1M 入力 / 64K 出力 / thinking 対応で、**動画・音声・PDF のネイティブ入力**が 3 社の中の最大の差別化点です。Pro 系のみ 200K トークン超で割増があります。Google Search グラウンディングの統合、Gemini API の無料枠(3.1 Pro を除く)も特徴です
- **コスト帯**(2026-07 時点の桁感): 入力単価は Flash-Lite から Pro(長文時)まで 10 倍超の幅(おおよそ $0.25〜$4 / 1M トークン)、出力はその約 5〜6 倍
- **注意が 2 つ**: ①「Flash = 廉価版」は旧世代の話です。現行の 3.5 Flash は主力(中位)で、廉価枠は Flash-Lite が担います ②最上位の 3.1 Pro は**プレビューのみで安定版がありません**(2026-07 時点)。本番の既定にする場合は安定版の Flash 系が基本です
- **補足**: knowledge cutoff が全モデル 2025-01 で据え置かれており、新しい知識は検索グラウンディングで補う想定とみられます

### オープンウェイト系

重みをダウンロードして自社ホストできるモデル群です。選ぶ動機(データ主権・カスタマイズ・コスト構造)と引き受ける責任は [OSS 系コーディングエージェントの議論](../08-coding-agents/open-source-coding-agents.md) と同型です。なお大型モデルの多くは MoE(Mixture of Experts: 推論のたびに一部のパラメータだけを起動する構造)です。

| ファミリー | 現行世代(2026-07 時点) | 最大公開モデル | ライセンス | コンテキスト長 |
| --- | --- | --- | --- | --- |
| Meta Llama | Llama 4(2025-04。以降 1 年超新世代なし) | Maverick 総 400B(MoE) | 独自 Community License | Scout 10M / Maverick 1M |
| Alibaba Qwen | Qwen3.5 / 3.6 | 総 397B(MoE) | Apache-2.0 | 262K(公式の明言は未確認) |
| DeepSeek | V4(Pro / Flash) | Pro 総 1.6T(MoE) | **MIT** | 1M |
| Mistral | Mistral 3 ファミリー | Large 3 総 675B(MoE) | Apache-2.0(オープン系) | 未確認(公式ページ参照) |
| OpenAI gpt-oss | gpt-oss-120b / 20b(2025-08) | 総 117B(80GB GPU 1 枚で動作) | Apache-2.0 | 131K |

押さえるべき論点は 4 つです。

1. **「オープンウェイト = 自由に商用可」ではない** — ライセンスは 3 類型あります: (a) Apache-2.0 / MIT(Qwen・DeepSeek・Mistral・gpt-oss。追加条件なし)、(b) 独自コミュニティライセンス(Llama。月間アクティブユーザー 7 億超は要個別契約 + 「Built with Llama」表示義務 + 派生モデルの命名義務)、(c) 独自利用規約(Google Gemma 等)。採用前にライセンス全文の確認が必須です
2. **大型帯は MoE が標準** — セルフホストのコストは総パラメータ(メモリ)とアクティブパラメータ(速度)の両方で見ます。「総 117B だが 80GB GPU 1 枚で動く」(gpt-oss-120b)のような設計が普通になった一方、DeepSeek V4 は「軽量版」の Flash でも総 284B でマルチ GPU 前提です
3. **推論はモード統合へ** — DeepSeek の R1(推論専用)路線は V4 の 3 段階思考モードに統合されました。「推論専用のオープンモデルを探す」より「思考モードを持つ汎用モデルを選ぶ」が 2026 年の基本形です
4. **「最上位だけクローズド」の分岐** — Qwen の最上位(3.7 Max)は重みの公開が確認できず API のみとみられ(2026-07 時点)、Mistral の最上位(Medium 3.5)は API 専用、Meta は最大モデル(Behemoth)未公開のまま停滞気味です。対照的に DeepSeek は 1.6T の最上位まで MIT で公開しています。「そのファミリーのオープン版は何段目までか」を確認して選びます

その他の勢力として、Google Gemma(小〜中型のオープンウェイト、独自規約)、xAI Grok(主力は API 専用、重み公開は旧世代のみ)、Amazon Nova(Bedrock 専用のプロプライエタリ)、中国系の Moonshot Kimi K2 系(1T 級)・Z.ai GLM 系(700B 級)の重み公開があります。

## 実務での注意点

### アンチパターン

- **本記事(または任意の紹介記事)だけで採用を決める** — カタログはスナップショットです。→ 最終確認日を見て、公式のモデル一覧・料金・非推奨ページで現況を再確認します
- **プレビュー・実験段階のモデルを本番の既定にする** — 仕様・価格・提供自体が変わりえます(2026-07 時点では Gemini 3.1 Pro が該当)。→ 本番は安定版(GA)+ ID ピン止めを基本にします
- **旧世代の常識で名前を解釈する** — 「Flash = 廉価」「o シリーズ = 推論の本命」「Llama = 実質自由ライセンス」はいずれも 2026-07 時点では不正確です。→ 名前ではなく現行の公式ポジショニングで判断します
- **退役スケジュールを見ずに新規開発する** — 提供終了が告知済みのモデル(Gemini 2.5 系など)の上に新規システムを作ると、すぐ移行作業が発生します。→ 非推奨ページの確認を選定手順に含めます
