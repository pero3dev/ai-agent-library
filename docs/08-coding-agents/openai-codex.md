# OpenAI Codex

## この記事の目的

OpenAI のコーディングエージェント Codex の製品構成(CLI / IDE 拡張 / クラウド / コードレビュー)・サンドボックスモデル・データ取り扱いを理解し、自分のチーム・用途に合うかを判断できるようになります。

- コーディングエージェントの候補として Codex を評価しているエンジニア
- ChatGPT を組織契約済みで、開発ワークフローへの展開を検討しているチーム

### 概要

Codex は OpenAI のコーディングエージェント製品群です。まず名称の注意から: 「Codex」という名前は 2021 年のコード生成**モデル**(非推奨化済み)にも使われていました。2025 年以降の Codex はエージェント**製品**の名称であり、別物です。古い記事・知識と混同しないでください。

2026 年時点の Codex は単一製品ではなく、同じエージェントを複数の面で提供する製品群です。5 分類([全体像](coding-agents-overview.md))では、ターミナル型(CLI は OSS)・IDE 統合型・クラウド実行型・GitHub 連携型のすべてに面を持ちます。特徴は次の 3 点です。

- **サンドボックス既定 + ネットワーク既定オフ** — ローカルでもクラウドでも、隔離とネットワーク遮断が既定側に倒された設計です
- **ChatGPT プランに同梱** — 既存の ChatGPT 契約(個人・組織)の範囲で使い始められます
- **AGENTS.md の本家** — ツール横断のルールファイル標準 AGENTS.md を最初から採用しています

### 提供形態と実行環境

| 面 | 内容 | 実行場所 |
| --- | --- | --- |
| Codex CLI | ターミナルの対話型エージェント。OSS(Apache-2.0、Rust 製) | ローカル |
| IDE 拡張 | VS Code 系(Cursor・Windsurf 含む)+ JetBrains。クラウドタスクの委任・監視も可能 | ローカル |
| Codex アプリ | デスクトップアプリ(macOS / Windows)。並列スレッド・git worktree・定期実行 | ローカル |
| Codex cloud | OpenAI 管理の隔離コンテナでタスク実行、PR として提出 | クラウド |
| コードレビュー | PR コメントで `@codex review`、または全 PR 自動レビュー | クラウド(GitHub 連携) |
| GitHub Action / `codex exec` / SDK | CI・自動化用の非対話実行、TypeScript / Python SDK | CI・任意 |
| Slack / Linear 連携 | メンションからクラウドタスクを起動 | クラウド |

- 対応 OS は macOS / Linux / Windows です
- クラウド実行は 2 フェーズ構成です: セットアップフェーズ(依存取得のためネットワークあり)→ エージェントフェーズ(**既定でオフライン**。ネットワークは明示的に有効化)。ローカルとクラウドで権限モデルが異なる点は導入時の確認事項です

### リポジトリ理解・編集・実行の仕組み

- **リポジトリ理解**: 事前の埋め込みインデックスを構築する記述は公式ドキュメントになく、オンデマンド探索型と判断できます(公式から推測)。AGENTS.md をセッション冒頭で取り込みます(1 ファイル既定 32 KiB 上限)
- **ファイル編集**: TUI 上で構文ハイライト付き diff をレビューできます。ロールバックの専用機構はなく、**通常の git ワークフローでレビュー・巻き戻す**設計です(公式明記)。デスクトップアプリは worktree で並列変更を分離し、diff 検査からコミット・プッシュまで行えます
- **コマンド実行**: サンドボックスモードと承認ポリシーの組み合わせで制御されます(次節)。非対話実行 `codex exec`、セッション再開 `codex resume`、`/review` によるローカル変更のレビューがあります

### 設定ファイルとカスタマイズ

- ルールファイルは **AGENTS.md**(オープン標準)。探索順はグローバル(`~/.codex/`)→ Git ルートから現在ディレクトリへ階層ごとに連結され、**近いファイルほど後に読まれて優先**されます。`AGENTS.override.md` による上書き機構もあります
- 設定は `~/.codex/config.toml`(個人)と `.codex/config.toml`(プロジェクト、信頼済みプロジェクトのみ)。`.codex/` に `config.toml`・`rules/`・`skills/` をコミットしてチーム標準化する方法が公式に案内されています
- 組織は **`requirements.toml`** を配布して、ChatGPT 認証のローカル各面(アプリ / CLI / IDE)にポリシーを強制適用できます(許可するサンドボックスモード・承認ポリシーの制限など)
- フック・プラグイン・スキル・サブエージェント(`config.toml` の `[agents]`)の拡張機構があります

### 権限管理とセキュリティ

権限は **2 軸の組み合わせ**で設計されています。

| 軸 | 選択肢 |
| --- | --- |
| サンドボックスモード(ファイル・ネットワーク境界) | `read-only` / `workspace-write`(既定。ワークスペース内の読み書き・実行可、外部とネットワークは承認要)/ `danger-full-access` |
| 承認ポリシー(いつ人に聞くか) | `untrusted`(安全な読み取りのみ自動)/ `on-request` / `never` |

- ローカルサンドボックスの実装は macOS = Seatbelt、Linux = bwrap + seccomp、**Windows = ネイティブ実装あり**(または WSL2)です
- **ネットワークは既定オフ**です。`workspace-write` でも明示的に有効化しない限り外部通信できません。[権限とセキュリティ](coding-agent-security.md) の「デフォルト拒否」原則が既定で体現されている構成です
- エスケープハッチ(`--yolo` 等)は公式自身が Elevated Risk と明記しています。使うなら隔離環境限定です
- **データ学習の既定はプランで異なります**: 個人プラン(Free/Go/Plus/Pro)は ChatGPT のトレーニングデータ設定に従い、学習に使われる場合があります(オプトアウト可。加えて Codex 独自の環境学習許可設定あり)。Business / Enterprise / Edu / API は既定で学習に使用しません。Enterprise は ZDR(Zero Data Retention)対応です
- 別機能として **Codex Security**(リポジトリの脆弱性スキャン)があります。エージェント本体のサンドボックス文書と名前が紛らわしいため区別してください

### 外部連携(MCP・CI・API)

- **MCP クライアント**: STDIO と Streamable HTTP に対応(CLI と IDE 拡張で公式明記。クラウド実行時の対応は未確認)。`codex mcp add` または `config.toml` で追加します
- Codex 自身を MCP サーバーとして公開する機能、ファーストパーティの Web 検索ツールも提供されています
- **GitHub 連携**: `@codex review` メンションによる PR レビュー(P0/P1 の問題に絞る方針で、変更ファイルに最も近い AGENTS.md のレビューガイドラインに従います)、GitHub Action による CI 組み込み
- **SDK**: TypeScript(Node.js 18+)/ Python(3.10+)。スレッド再開・サンドボックスレベル指定が可能です

### チーム導入と提供プラン

- Codex は ChatGPT の各プラン(Free / Go / Plus / Pro / Business / Edu / Enterprise)に含まれます(Free / Go は限定的)。API キー認証による従量課金も選べます
- 利用制限は「5 時間ウィンドウあたりのメッセージ数」構造で、超過分はクレジット購入で継続できます。具体値は変動が激しいため公式料金ページ(参考資料)で確認してください
- 組織管理: ワークスペース設定で Codex Local / cloud の利用可否を制御、`requirements.toml` による集中ポリシー、Analytics API(利用状況)、Compliance API(ログ・タスクの追跡)が提供されます

### 代表的なユースケースと向き不向き

**公式が想定する用途**: PR の事前レビュー(人のレビュー前の回帰検出)、リファクタリング・デッドコード除去、大規模コードベースの理解、コード移行、ドキュメント最新化、脆弱性バックログの修正、クラウドでのデプロイまでの一気通貫などです。

**向き不向き(特性として)**:

- 向く: ChatGPT を組織契約済みのチーム(追加契約なしで開始可能)、サンドボックス・ネットワーク遮断を既定にしたい組織、AGENTS.md でマルチツール標準化を進めたいチーム、CLI の OSS 性(挙動の検証可能性)を重視する場合
- 注意が要る: OpenAI モデル前提のため、モデル選択の自由度を最優先する場合は BYOK 型の検討が必要です。クラウド実行は GitHub 連携が前提の設計です。個人プランではデータ学習の既定設定の確認が必須です

## 実務での注意点

### アンチパターン

- **2021 年の「Codex モデル」の知識で語る** — 現行のエージェント製品とは別物で、能力・仕組み・提供形態のすべてが異なります。→ 2025 年以降の公式ドキュメントだけを根拠にします
- **クラウドとローカルの権限モデルを同一視する** — ローカルはサンドボックスモード × 承認ポリシー、クラウドは 2 フェーズ(セットアップのみネット可)と、境界の引き方が違います。→ 面ごとにセキュリティレビューします
- **`--yolo` での常用** — サンドボックスと承認の両方を外した状態は公式自身が高リスクと明記しています。→ 使い捨てコンテナ内に限定します
