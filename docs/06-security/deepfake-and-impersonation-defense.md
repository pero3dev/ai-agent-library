# ディープフェイク・なりすましへの防御

## この記事の目的

音声クローン・映像合成による**なりすまし**(送金詐欺・虚偽指示)への、組織的な防御を設計できるようになります。脅威の類型を押さえ、「防御の中心は検出でなく**プロセス**」という原則のもと、本人確認の設計・重要指示のコールバック検証・合言葉・支払いプロセスの二重化・従業員訓練・通報経路・経営層/ブランドのなりすまし監視を持ち帰れる状態を目指します。本記事は**防御側**に徹し、ディープフェイクの作成手法には触れません。

## 対象読者

- 送金・機密指示・顧客対応などで、なりすましのリスクにさらされる組織のセキュリティ・業務プロセス担当
- 経営層・ブランドのなりすまし対策を設計する立場の人

## 前提知識

- [生成物の来歴と検出](content-provenance-and-detection.md) — 検出の限界(なぜプロセスで守るか)
- [Agent の脅威モデル概観](threat-model-overview.md) — 脅威の全体分類(本記事はなりすましの各論)

## 本文

### 概要: 「見破る」より「プロセスで止める」

音声クローン・映像合成の高度化で、**声や顔は「本人である証拠」にならなくなりつつあります**。公的機関も「AI 生成は見分けが困難」と注意喚起しています。したがって防御の中心は、「偽物を見破る(検出)」ではなく、**「なりすましが成立してもプロセスで被害を止める」**ことに移ります。本記事はそのプロセス設計を扱います。作成手法には触れません。

### 脅威類型

なりすましの脅威は、おおむね次の類型です。

| 類型 | 内容 |
| --- | --- |
| 音声クローンによる電話詐欺 | 経営者・家族・取引先の声を装い、緊急の送金・情報を要求する |
| ビデオ会議のなりすまし | 合成した顔・声でビデオ会議に現れ、指示・承認を装う |
| 偽コンテンツによる風評 | 経営層・ブランドを装った偽の発言・広告で信用を毀損する |

- 共通する手口は、**本人らしい声・顔で、緊急の送金や機密の受け渡しを迫る**ことです。「急いで」「内密に」という圧力で、確認プロセスを飛ばさせようとします
- 音声クローンの**自社利用の統制**(本人同意・悪用対策)は [音声合成(TTS)と声の設計](../12-multimodal/speech-synthesis-and-voice-design.md) が扱います。本記事は**外部からのなりすまし攻撃への防御**です

### 防御の中心はプロセス

防御の核心は、**「声・顔を本人確認に使わない」前提への移行**と、**プロセスによる検証**です。

- **声・顔を本人確認に使わない**: 「声が本人だから」「ビデオに映っているから」を本人確認の根拠にしません。これらは合成できる前提に立ちます
- **重要指示のコールバック検証**: 送金・機密・権限変更などの重要指示は、**別の信頼できる手段でかけ直して本人確認**します(公的機関が共通して推奨する第一の防御)。受けた連絡先でなく、**既知の正規の連絡先**にかけ直します
- **合言葉・第二経路**: 家族・組織内で**合言葉(code word)**を決める、指示を**第二の経路**(別の担当者・別のシステム)でも確認する、といった多重化をします
- **支払いプロセスの二重化**: 送金・支払いは、単独の指示で実行できないようにします。**複数人の承認**([Human-in-the-Loop 設計](../02-architecture/human-in-the-loop.md))・二重チェック・上限の設定で、1 つのなりすましで被害が出ない構造にします
- **急かしに応じない運用**: 「急ぎ」「内密」を理由に確認を飛ばさない、と運用で決めます。緊急性の演出こそが攻撃の常套手段です

### 検出技術の位置づけ

ディープフェイク検出ツールは、**補助であり、単独で頼りません**([生成物の来歴と検出](content-provenance-and-detection.md)の検出の限界)。

- **単独で頼らない**: 検出は誤判定(偽陽性・偽陰性)を伴い、回避との軍拡の中にあります。「検出ツールが本物と言ったから本物」とはしません
- **プロセスの補助として**: 検出は、プロセス(コールバック検証・多重承認)を補う一要素です。プロセスを検出に置き換えるのでなく、プロセスに検出を足します
- **来歴も補助**: 来歴([生成物の来歴と検出](content-provenance-and-detection.md))も、あれば手がかりですが、剥がれうるため単独の根拠にはしません

### 従業員訓練と通報経路

技術・プロセスに加え、**人の備え**が要ります。なりすましは人を狙う攻撃です。

- **従業員訓練**: 「声・顔は偽装されうる」「急かし・内密は危険信号」「重要指示はかけ直して確認」を、訓練で共有します。とくに送金・機密を扱う担当は重点的に
- **通報経路を明確に**: 「怪しい」と思ったときに、迷わず相談・通報できる経路を用意します。「確認したら失礼では」という心理が確認を妨げるため、**確認は正当**という文化を作ります
- **インシデントとして扱う**: なりすまし被害・未遂は、[インシデント対応](../05-operations/incident-response.md)として記録・対応します

### 経営層・ブランドのなりすまし監視

自組織の**経営層・ブランドを装ったなりすまし**への監視も要ります。

- **経営層のなりすまし**: 経営者を装った偽の指示・発言は、対内(送金指示)にも対外(偽の声明)にも使われます。経営層になりすました連絡には、上記のコールバック検証を徹底します
- **ブランドのなりすまし監視**: 自社ブランド・著名人を装った偽広告・偽アカウントを監視し、見つけたら通報・削除申請のプロセスを持ちます
- **公的機関の注意喚起を参照**: 手口・対策は、公的機関(下記参考資料)が継続的に注意喚起しています。最新の手口を運用に反映します

## 実務での注意点

### アンチパターン

- **声・顔を本人確認の根拠にする** → 音声クローン・映像合成で突破される → 声・顔は偽装前提とし、別手段で本人確認する
- **受けた連絡先にそのままかけ直す** → 攻撃者の用意した連絡先で「確認」してしまう → 既知の正規の連絡先にかけ直す
- **単独の指示で送金できる** → 1 つのなりすましで被害が出る → 複数人承認・二重チェック・上限で多重化する
- **検出ツールを本人確認の決め手にする** → 誤判定で被害を通す/正当な相手を拒む → 検出は補助とし、プロセスで守る
- **「急ぎ・内密」で確認を飛ばす** → 緊急性の演出という攻撃の常套手段に乗る → 急かしに応じず、確認を正当とする文化を作る

### チェックリスト

- [ ] 声・顔を本人確認の根拠にしない前提に移行した
- [ ] 重要指示は既知の正規連絡先へのコールバックで本人確認している
- [ ] 合言葉・第二経路など、確認の多重化を用意した
- [ ] 送金・支払いを複数人承認・二重チェック・上限で多重化した
- [ ] 検出・来歴は補助と位置づけ、単独の根拠にしていない
- [ ] 従業員訓練で「偽装されうる・急かしは危険・かけ直して確認」を共有した
- [ ] 迷わず相談・通報できる経路を用意し、確認を正当とする文化を作った
- [ ] 経営層・ブランドのなりすまし監視と対処プロセスを持っている
