# rag-basics — 依存最小の RAG(Python)

RAG の骨格「**取り込み → 検索 → 根拠付き生成**」を、ベクトル DB も重いフレームワークも使わず、純 Python で示すサンプルです。次のドキュメントの内容を実装しています。

- [RAG 実装パターン](../../../docs/03-implementation/rag-implementation-patterns.md) — 検索と生成の分離、根拠(出典)提示

## このサンプルの要点

- **埋め込みは文字 bigram で代用**: 実運用では埋め込みモデルを使いますが、ここでは依存を増やさないため、文字 2-gram の頻度ベクトルで近さを測ります(`docs.json` の小さな文書集合が対象)。「ベクトル化して近さで引く」考え方は本物と同じです
- **検索と生成を分ける**: 検索(純 Python・コサイン類似度で上位 k 件)と生成(LLM)を分離し、検索結果を出力して**根拠が見える**ようにしています
- **資料だけを根拠に答えさせる**: プロンプトで「資料になければ分からないと答える」よう指示し、出典 id を付けさせます(幻覚対策)

## 前提

- Python 3.11 以降(`--mock` と検索は追加依存なし・標準ライブラリのみで動きます)
- 実 API 実行時のみ: Anthropic API キー(環境変数 `ANTHROPIC_API_KEY`)と `pip install -r requirements.txt`

## 実行

```bash
# モック実行(API キー不要・依存インストール不要)
python rag.py --mock "経費の申請締切はいつですか?"

# 実 API 実行(要 ANTHROPIC_API_KEY)
pip install -r requirements.txt
python rag.py "経費の申請締切はいつですか?"
```

検索結果(どの文書が上位に来たか・スコア)が表示され、その資料を根拠に回答されます。
