## 契約で確認すべき観点(列挙)

記事では「契約実務で確認すべき観点の列挙」までに徹し、具体条文の解釈はしない。生成モデル各社の IP 補償・出力所有権の一次情報は [`research/multimodal/generation.md`](../multimodal/generation.md)(2026-07-08)が正。

1. **モデル提供者の IP 補償条項(IP indemnification)の有無と範囲** — 学習データ由来の第三者著作権侵害クレームを提供者が防御・補償するか、上限(cap)・除外(禁止用途・改変・入力素材起因等)・対象製品層(GA/beta、無料/有料/Enterprise)を確認する観点。generation.md に記載のある実例: Amazon Nova(AWS Service Terms §50.10 で GA 出力に無上限=uncapped の IP 補償)、Adobe Firefly(Enterprise/API 文脈で提供)、Luma(ToS で IP 侵害クレームの防御・補償に言及)。各社条件は改定頻繁のため要再確認
2. **学習データの出所・素材ライセンスの表明** — 提供者が学習データの適法な取得・ライセンス素材使用を表明しているか(例: generation.md 記載の Adobe Firefly の「Adobe Stock・オープンライセンス・パブリックドメインで学習」との自己申告)。表明は事実確認の対象であり保証内容は契約で確認する観点
3. **成果物(出力)の権利帰属・所有権条項** — 出力の所有権がユーザーに帰属するか、利用範囲(商用可否・再配布)や解約後の継続利用が規約でどう定められるか(例: generation.md 記載の Stability・Midjourney・Runway は「出力はユーザー所有」旨。ただし年商閾値等の条件が付く場合あり)
4. **入力素材(顧客提供データ)の権利・責任分担** — ユーザーが入力する素材の権利処理責任、入力データの学習利用の既定(compliance.md の各社データ取扱いが正)、秘密保持・データ保持の観点
5. **禁止用途・利用ポリシー(Prohibited Use / Usage Policy)** — 補償や権利付与が禁止用途違反時に失効しうる点を確認する観点
6. **準拠法・紛争解決・通知窓口** — 著作権者からの申立て窓口(EU Art. 53 の権利者向けコンタクトポイント等)や、クレーム発生時の連絡・防御手続の定めを確認する観点

> **方針:** これらは「契約で **確認すべき論点** の列挙」であり、各条項の有利/不利や適法性の評価はしない。実際の条件は各社の Terms / Service Terms / DPA と、必要に応じて弁護士確認に委ねる。
