# 13-domain-agents — ドメイン別エージェント設計

代表的な応用ドメイン(リサーチ・データ分析・RPA 代替・アシスタント・検索・執筆翻訳・教育ほか)ごとに、**そのドメイン固有の設計判断**を体系化するセクションです。01〜06・10 章がドメイン非依存の「作り方・運用の仕方」を扱うのに対し、本セクションは「このドメインでは何が効き・何で失敗するか」という横の体系を扱います。

- **置くもの**: ドメインごとの特性分析(検証可能性・失敗コスト・データアクセス)、定番アーキテクチャの型、ドメイン固有の落とし穴、評価の設計
- **置かないもの**: 各ドメインの業務知識そのもの(統計学・翻訳理論・教育学の体系 → 設計に効く範囲の参照まで)、特定製品の導入手順(類型と判断軸まで)、規制ドメインの内容解説(→ [業界別規制の入口マップ](../09-business/industry-regulations-map.md) への接続まで)

> **ドメイン非依存の作り方は各章が正本です**: 各記事は共通論(ループ・RAG・権限・HITL・評価)を再説明せず、[03-implementation](../03-implementation/README.md)・[04-evaluation](../04-evaluation/README.md)・[06-security](../06-security/README.md) 等へリンクで委譲します。また [07-case-studies](../07-case-studies/README.md) が特定シナリオを「縦の物語」で通すのに対し、本セクションは「横の設計ガイド」です(相互リンク)。案件を企画・推進する視点は [09-business](../09-business/README.md) の [ユースケース発見](../09-business/usecase-discovery.md) が正本で、本セクションはその「向き不向き」判断のドメイン別の具体化として機能します。
