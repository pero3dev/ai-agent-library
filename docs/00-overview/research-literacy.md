# AI 情報の追い方(一次情報の目利き)

## この記事の目的

変化の速い AI 情報を、**振り回されずに追う個人・チームの仕組み**を作れるようになります。情報源の階層(一次・準一次・二次)の見分け方、論文の実務的な読み方、ハイプの見分け方、そして「追わなくてよいものを決める」判断を持ち帰ります。これは本ライブラリ自身が実践している調査方法論(`research/` の一次情報メモ、各記事の「変わりやすい項目」)を、読者が自分で使える形に一般化したものです。

## 対象読者

- 次々に出る AI のニュース・論文・ツールに、追いきれない・振り回されると感じているエンジニア・チーム
- チームで「何を・どう追うか」の情報の仕組みを作りたいテックリード

## 前提知識

- [AI Agent 学習ロードマップ](learning-roadmap.md) — 「何をどの順で学ぶか」(本記事は「どう情報を追い続けるか」)
- 特別な前提はありません

## 本文

### 概要: 「全部追う」をやめる

AI 情報は、全部を追おうとすると破綻します。健全な情報の付き合い方は、**情報源の質を見分け、追う価値のあるものを絞り、原理と具体を分けて投資する**ことです。ニュースの洪水に流されるのではなく、**自分の仕組み**を持つのが目標です。本ライブラリも、各トピックについて一次情報を `research/` にメモし、変化の速い箇所に「変わりやすい項目」を残す、という仕組みで鮮度と正確さを両立させています。その方法を開示します。

### 情報源の階層

情報は、**発信源からの距離**で質が変わります。3 階層で見分けます。

| 階層 | 何か | 例 | 信頼の目安 |
| --- | --- | --- | --- |
| 一次情報 | 発信源そのもの | 論文(原典)・公式ドキュメント・公式ブログ・規格本文・法令 | 最も信頼できる。根拠にするならここ |
| 準一次情報 | 第三者による直接の実測・検証 | 独立した再現実験・第三者ベンチマーク・実測レポート | 一次を補完・検証する。誰がどう測ったかを見る |
| 二次情報 | 一次・準一次の解説・要約 | まとめ記事・ニュース・解説動画・SNS | 「存在を知る」までに使う。**根拠には使わない** |

原則は、**根拠は一次情報に置く**ことです。二次情報は「何が起きたかを知る入口」として便利ですが、旧版ベースだったり、伝言ゲームで歪んでいたりします。二次情報で見つけたら、一次情報まで遡って確かめてから使います(本ライブラリの `research/` メモは、この「一次まで遡った記録」です)。

### 論文の実務的な読み方

論文を精読する時間はなくても、**実務判断に使う読み方**があります。全部を数式まで追う必要はありません。

- **アブストラクト → 図表 → 限界節**の順で読む。主張(アブスト)、証拠(図表)、著者自身が認める弱点(限界・Limitations)を押さえれば、実務での使いどころは判断できます
- **再現・検証の有無を見る**: コード・データが公開されているか、第三者が再現できているか。再現されていない華々しい結果は、割り引いて読みます
- **自己申告と査読を区別する**: 査読前のプレプリントや、企業の自社報告は、それとして扱います(誤りではないが、独立検証を待つ姿勢)
- **「限界節」を軽視しない**: 良い論文ほど限界を正直に書きます。そこに実務適用時の落とし穴が書かれています

ベンチマークの数字の読み方(ハーネス依存・汚染・飽和)は[エージェントベンチマークの全体像](../04-evaluation/agent-benchmarks-landscape.md)が正本で、その姿勢は論文全般に一般化できます。

### ハイプの見分け方

AI はハイプ(過剰な期待)が生まれやすい領域です。冷静に見分ける観点を持ちます。

- **デモと製品の距離**: 印象的なデモは、うまくいく条件で最適化されています。「デモが動く」と「本番で使える」の間には大きな距離があります([ケーススタディ: 撤退した PoC](../07-case-studies/case-study-failed-poc.md)の「デモの罠」)
- **自己報告の数値を割り引く**: 「精度 X%」「世界初」「完全自律」といった自己申告の主張は、測定条件・独立検証の有無を確認するまで断定に使いません
- **主語と出典を確認する**: 「誰が」「どこで」言ったか。伝聞の伝聞は、途中で条件が抜け落ちます
- **極端な予測ほど当たりにくい**: 「N 年で X が消える/実現する」式の断定的予測は、根拠が弱いまま不安や期待を煽ります([AI 時代のエンジニアキャリア戦略](../15-human-ai/ai-career-strategy.md)のハイプとの付き合い方)

見分けの核心は、**「印象」ではなく「検証可能な根拠」で判断する**ことです([AI 出力の検証習慣](../15-human-ai/verifying-ai-outputs.md)の情報源版)。

### 追いかける仕組み

追うべきものは、**個人の記憶でなく仕組み**で追います。本ライブラリの「変わりやすい項目(定点観測)」運用が、そのままチーム実践になります。

- **定点観測を決める**: 「何を・どの一次情報源で・どの頻度で見るか」を明文化する(例: 四半期ごとに主要モデルの公式ページを確認)。担当と周期を決めると属人化を防げます
- **鮮度を記録する**: 「いつ・何を確認したか(最終確認日)」を残す。古くなった情報を検知できます
- **更新起点を持つ**: 変化の速いテーマは、`research/` のような一次情報メモを更新の起点にする。毎回ゼロから調べ直さずに済みます
- **チームで共有する**: 誰かが追った結果を共有し、全員が同じものを重複して追わないようにします

### 追わなくてよいものを決める

最も重要なのは、**追わないものを決める**ことです。すべてを追うのは不可能で、その必要もありません。

- **原理は安定、具体は変わる**: [Agent ループ](../01-concepts/agent-loop.md)・[LLM の能力と限界の由来](../10-llm-foundations/capabilities-and-limits.md)といった原理は変わりにくく、一度理解すれば長く効きます。一方、モデル名・価格・フレームワークの API は速く変わります
- **投資を二層に分ける**: 原理は「長く効かせるために」じっくり学び、具体は「今使うために」都度キャッチアップする。この使い分けが、情報疲れを防ぎます
- **自分の仕事に効くかで絞る**: 話題性でなく、自分の業務・判断に影響するかで追う対象を選びます。関係の薄い最新動向は「存在を知る」までで十分です

本ライブラリが「概念(01)・LLM 基礎(10)を飛ばすな」と繰り返すのは、**原理への投資が最も減価しにくい**からです([学習ロードマップ](learning-roadmap.md))。

### この理解が効く場面

- **情報の取捨選択**: ニュースの洪水から、一次情報で裏取りする価値のあるものを見分ける
- **論文・技術報告の判断**: 実務に使えるか(限界・再現性)を短時間で見積もる([エージェントベンチマークの全体像](../04-evaluation/agent-benchmarks-landscape.md))
- **チームの情報運用**: 定点観測・鮮度管理の仕組みを作り、属人化と情報疲れを防ぐ
- **学習投資の配分**: 原理と具体を分け、減価しにくい方に軸足を置く([スキルマップ](skill-map.md)・[AI 時代のエンジニアキャリア戦略](../15-human-ai/ai-career-strategy.md))

## 実務での注意点

### アンチパターン

- **二次情報(まとめ記事・SNS)を根拠に判断する** → 旧版ベース・伝言ゲームで歪み、誤った判断をする → 一次情報まで遡って確かめてから根拠にする
- **自己報告の数値・「世界初」を額面で受け取る** → 測定条件・独立検証を確認せず、ハイプに乗る → 検証可能な根拠(再現・第三者実測)を確認するまで断定に使わない
- **全部を追おうとする** → 情報疲れで破綻し、かえって重要なものを見落とす → 追わないものを決め、自分の仕事に効くかで絞る
- **具体(モデル名・API)ばかり追い、原理を後回しにする** → 具体はすぐ古くなり、投資が短命に終わる → 原理と具体を分け、原理に軸足を置く
- **情報追いを個人の記憶任せにする** → 属人化し、古い情報に気づけない → 定点観測・最終確認日の記録という仕組みで追う

### チェックリスト

- [ ] 情報源を一次・準一次・二次で見分け、根拠は一次情報に置いている
- [ ] 二次情報で見つけたら、一次まで遡って確かめている
- [ ] 論文を「アブスト → 図表 → 限界節」で読み、再現・検証の有無を見ている
- [ ] 自己報告の数値・断定的予測を、検証可能な根拠を確認するまで割り引いている
- [ ] 「何を・どこで・どの頻度で」追うかの定点観測を明文化した
- [ ] 追わないものを決め、原理と具体で学習投資を二層に分けている
- [ ] チームで情報を共有し、重複した追いかけを避けている
