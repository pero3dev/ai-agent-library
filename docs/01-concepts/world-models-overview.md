# 世界モデルの概観

## この記事の目的

「世界モデル(world model)」という語が指す範囲を整理し、それがエージェントとどう関わるかを掴めるようになります。バズワードとして一括りにされがちなこの語を、**3 つの用法**に分けて理解し、ソフトウェアエージェントの実務(特にシミュレーション評価)への含意まで見通せる状態がゴールです。

## 対象読者

- 「世界モデル」という語を目にするが、何を指しているのか整理したいエンジニア
- エージェントの訓練・評価に生成的シミュレーションが使えるか関心のある人

## 前提知識

- [AI Agent とは何か](what-is-an-ai-agent.md) — 「観測 → 思考 → 行動」の骨格(世界モデルはその「行動の結果」を扱う)
- [フィジカル AI とロボティクスの概観](physical-ai-overview.md) — 世界モデルが最も使われている物理世界の文脈

## 本文

### 概要: 「世界モデル」は 1 つの意味ではない

「世界モデル」は文脈によって別のものを指します。まず用法を分けることが理解の第一歩です。本記事は概念の整理を担い、物理世界での具体は [フィジカル AI とロボティクスの概観](physical-ai-overview.md) が、エージェントを評価する環境の実装は [エージェント評価環境の構築](../04-evaluation/evaluation-environments.md) が扱います。

### 世界モデルの 3 つの用法

| 用法 | 何をするか | 例 |
| --- | --- | --- |
| ①環境の生成的シミュレーション | 環境そのものを生成し、その中を動けるようにする | 操作可能な仮想環境をリアルタイムに生成するモデル |
| ②行動結果の予測 | 「この行動を取ったら次にどうなるか」を予測する | ロボットが動く前に結果を頭の中で試す内部モデル |
| ③エージェント訓練・評価のインフラ | 学習・評価に使うデータや環境を製造する基盤 | 合成データ生成・シミュレータとしての世界モデル |

「世界モデル」と聞いたら、**このどれを指しているか**を確認します。①は生成 AI 寄り、②はエージェントの内部表現、③は学習インフラで、関わり方がまったく違います。

### なぜ注目されるのか

世界モデルが注目される理由は、エージェント学習の 2 つの壁に効くからです。

- **データの製造**: エージェント(とくに物理世界の)を訓練するには大量の経験データが要りますが、Web テキストのような大規模データが存在しません。世界モデルは、シミュレーションで**合成データを製造**することで、この不足を埋めます([フィジカル AI とロボティクスの概観](physical-ai-overview.md)のデータ「製造」の文脈)
- **安全な試行錯誤**: 実世界で失敗を繰り返すのは危険・高コストです。世界モデルの中でなら、**安全に試行錯誤**でき、行動の結果を実行前に予測できます

要点は、世界モデルが「賢い個体そのもの」ではなく、**賢い個体を作るための工場・練習場**として先に実用化しつつあることです。

### ソフトウェアエージェントへの含意

物理世界の話に見えて、ソフトウェアエージェントにも接点があります。

- **シミュレーション評価**: 「環境を用意してエージェントを動かし、結果を検証する」という発想は、ソフトウェアエージェントの評価環境と同じです([エージェント評価環境の構築](../04-evaluation/evaluation-environments.md)の状態付きサンドボックス)。世界モデルは、この評価環境を**生成的に作る**方向の延長線上にあります
- **行動結果の予測**: 「実行前に結果を予測して、危険な行動を避ける」という発想は、不可逆な操作を扱うエージェントの設計([Human-in-the-Loop 設計](../02-architecture/human-in-the-loop.md))とも通じます
- ただし、ソフトウェアの世界では「本物の環境(サンドボックス)を用意する」ほうが忠実で確実なことが多く、生成的な世界モデルが実務の主役になっているわけではありません

### 現在地の読み方: デモと実用の距離

世界モデルは話題性が高く、デモが派手なため、**現在地を冷静に読む**ことが要ります。

- 2026 年時点では、世界モデルは「エージェントを直接動かすモデル」よりも、**③の学習・評価インフラ**としての実用化が先行しています(データ工場・シミュレータ)
- 派手な「操作できる生成環境」のデモは、一貫性・忠実度・実用スケールの点で、まだ研究とデモの域を出ない部分が大きいです
- 個別モデルの顔ぶれ・性能は変化が速く、横並びの共通ベンチマークも確立していません(下記 TODO)。「何ができるようになったか」は、デモではなく用途(③のインフラか、①②の実用か)で読みます

## 実務での注意点

### アンチパターン

- **「世界モデル」を 1 つの意味だと思い込む** → 生成シミュレータ・行動予測・学習インフラを混同し、議論が噛み合わない → まず 3 つの用法のどれかを特定する
- **デモの派手さで実用度を判断する** → 操作できる生成環境のデモを実用と誤認する → 用途(学習・評価インフラか、実用の個体か)と一貫性・スケールで読む
- **ソフトウェア評価に生成的世界モデルを急いで持ち込む** → 本物のサンドボックスの方が忠実・確実なのに複雑化する → まず状態付きサンドボックス([エージェント評価環境の構築](../04-evaluation/evaluation-environments.md))で足りるかを見る

### チェックリスト

- [ ] 「世界モデル」がどの用法(生成シミュレーション・行動予測・学習インフラ)を指すか特定した
- [ ] その世界モデルが「個体」なのか「学習・評価インフラ」なのかを区別している
- [ ] デモの派手さではなく用途と成熟度で現在地を読んでいる
- [ ] ソフトウェア評価では、生成的世界モデルより本物のサンドボックスで足りないかを先に検討した
