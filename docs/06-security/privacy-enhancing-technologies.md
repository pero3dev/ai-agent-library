# プライバシー強化技術の概観

## この記事の目的

マスキングの先にあるプライバシー強化技術(PET: Privacy-Enhancing Technologies)— 仮名化・匿名化・差分プライバシー・連合学習・秘密計算 — の「**何が実務で現実的か**」の見取り図を持てるようになります。各技術の階層・成熟度・適用場面を理解し、「高度な技術より先にやること」を踏まえた選定の判断フレームを持ち帰れる状態を目指します。会話データの収集・マスキングの実装は [会話データの管理基盤](../05-operations/conversation-data-management.md) が正本で、本記事は**技術の見取り図**に絞ります。数理の詳解はせず、適用判断に必要な直感まで扱います。

## 対象読者

- 個人情報・機微データを扱う AI システムで、マスキングの先の選択肢を検討するセキュリティ・データ基盤担当
- 「差分プライバシー」「連合学習」などの語を目にするが、自社で現実的かを見極めたいテックリード

## 前提知識

- [会話データの管理基盤](../05-operations/conversation-data-management.md) — PII 検出・マスキング・保持/削除の実装(本記事はその先の技術の見取り図)
- [AI のためのデータガバナンス](../05-operations/data-governance-for-ai.md) — データの分類・オーナーシップ(何を守るかの前提)

## 本文

### 概要: 「高度な技術」の前に見取り図を

「プライバシーを守る = 差分プライバシーや秘密計算を導入する」と飛躍しがちですが、実務では**技術に階層があり、多くの場合もっと基本的な手段で足ります**。本記事は PET を階層で整理し、それぞれの成熟度・適用場面と、「高度な技術より先にやること」を示します。会話データの実装は [会話データの管理基盤](../05-operations/conversation-data-management.md) が正本です。

### 技術の階層

PET は、おおむね次の階層で捉えられます。上ほど基本的で広く使え、下ほど高度で適用場面が限られます。

| 階層 | 技術 | 何をするか |
| --- | --- | --- |
| ①検出・マスキング | PII 検出・マスキング | 個人情報を見つけて隠す・置き換える |
| ②仮名化・匿名化 | 仮名化・匿名化 | 個人を特定できないよう加工する(再識別の限界あり) |
| ③統計的保護 | 差分プライバシー | 集計・出力に「個人の影響」が残らないようにする |
| ④分散学習 | 連合学習 | データを集めずに、分散したまま学習する |
| ⑤秘密計算 | 秘密計算・TEE | データを暗号化・隔離したまま計算する |

- **①検出・マスキング**: 最も基本で広く使われます([会話データの管理基盤](../05-operations/conversation-data-management.md))。個人情報を検出して隠す・置き換えます。多くのユースケースはここで足ります
- **②仮名化・匿名化**: 個人を特定できないよう加工します。ただし**再識別の限界**があります — 他のデータと突き合わせると個人が特定できてしまうことがあり、「匿名化したから安全」とは限りません
- **③差分プライバシー**: **差分プライバシー(differential privacy)** — 集計や出力に「ある個人が含まれるかどうか」の影響がほぼ残らないようにする統計的な保護 — です。「この統計から特定の個人の情報を逆算できない」ことを保証する方向の技術です。**精度とのトレードオフ**(プライバシーを強めるほど結果の精度が落ちる)があります
- **④連合学習**: **連合学習(federated learning)** — データを 1 か所に集めず、各所に置いたまま学習し、モデルの更新分だけを持ち寄る — 手法です。「生データを外に出さずに学習したい」場面に効きます
- **⑤秘密計算・TEE**: データを暗号化したまま計算する秘密計算や、隔離された実行環境(TEE)で処理する技術です。最も強力ですが、性能・実装のコストが高く、適用は限られます

### 成熟度と適用場面

各技術は、「**集計・学習・推論のどこに効くか**」で使い分けます。

- **集計・分析**: 統計・レポートを出すなら、差分プライバシーが「集計から個人が逆算されない」保証に効きます
- **学習**: 生データを集められない(規制・データ主権)なら、連合学習で分散したまま学習する選択があります([ファインチューニングと蒸留](../03-implementation/fine-tuning-and-distillation.md)の学習側)
- **推論・処理全般**: データを暗号化・隔離したまま扱う必要があるなら、秘密計算・TEE が候補ですが、コストが高く適用は限定的です
- **成熟度は一様でない**: 検出・マスキング・匿名化は広く実用され、差分プライバシーは集計で実用例があり、連合学習・秘密計算は適用場面が特定的です。「新しくて強力」でも、自社の場面で現実的かは別です

### 高度な技術より先にやること

PET の実務で最も重要な原則は、**高度な技術に飛びつく前に、基本を尽くす**ことです。

- **データ最小化**: そもそも**集めない・持たない・保持しない**のが最強のプライバシー保護です。差分プライバシーを議論する前に、「このデータは本当に要るか」を問います([AI のためのデータガバナンス](../05-operations/data-governance-for-ai.md))
- **アクセス統制**: 集めたデータへのアクセスを、必要な人・用途に絞ります([マルチテナント設計とテナント分離](../02-architecture/multi-tenancy-and-isolation.md)・[会話データの管理基盤](../05-operations/conversation-data-management.md))。高度な暗号技術より、まず「誰が見られるか」を絞るほうが効きます
- **検出・マスキング**: 個人情報の検出・マスキングを確実にします。多くの要件はここまでで満たせます
- **順序を守る**: 「データ最小化 → アクセス統制 → 検出・マスキング」を尽くしてから、差分プライバシー・連合学習・秘密計算を検討します。順序を飛ばすと、コストばかり高く効果の薄い対策になります

### 選定の判断フレーム

PET を選ぶときの判断は、次の順で行います。

1. **要件を特定する**: 何を・誰から・なぜ守るか(規制要件・データ主権・機微度)を明確にします
2. **基本で足りるか**: データ最小化・アクセス統制・マスキングで要件を満たせないかを、まず検討します
3. **場面に合う技術を選ぶ**: 足りない場合、集計なら差分プライバシー、集められない学習なら連合学習、と場面に合わせます
4. **コストと現実性を測る**: 精度低下・性能コスト・実装/運用の負担を見積もり、現実的かを判断します。「導入できるが割に合わない」を避けます

## 実務での注意点

### アンチパターン

- **高度な PET に飛びつく** → コストばかり高く、基本(最小化・統制)が抜ける → データ最小化 → アクセス統制 → マスキングを先に尽くす
- **「匿名化したから安全」と考える** → 他データとの突き合わせで再識別される → 再識別の限界を前提に、アクセス統制と併用する
- **差分プライバシーの精度低下を見ずに導入する** → 結果の有用性が落ちて使えない → プライバシーと精度のトレードオフを測って判断する
- **場面に合わない技術を選ぶ** → 集計問題に連合学習を持ち込む等のミスマッチ → 集計・学習・推論のどこに効くかで選ぶ
- **要件を特定せず技術ありきで進める** → 守るべきものが曖昧なまま投資する → 何を・誰から・なぜ守るかを先に特定する

### チェックリスト

- [ ] 守るべき要件(規制・データ主権・機微度)を特定した
- [ ] データ最小化(そもそも集めない・持たない)を先に検討した
- [ ] アクセス統制で「誰が見られるか」を絞っている
- [ ] 検出・マスキングを確実にしている
- [ ] 匿名化の再識別の限界を理解し、過信していない
- [ ] 高度な PET は、場面(集計/学習/推論)に合うものを選んでいる
- [ ] 精度低下・性能・運用コストを測り、現実的かを判断した
